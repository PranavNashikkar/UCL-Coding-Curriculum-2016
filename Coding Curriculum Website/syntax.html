1. Syntax

1.1 Importance of Syntax



When we look at Java source code we have to follow a set of rules in order for the code to be identified as Java and not another programming language. Moreover, these rules allow a compiler to convert this source code into a working program. A compiler is a program that processes source code and turns into actual machine code that the computer understands and can run. If these rules are not followed, it will lead to error known as a Syntax error.


1.2 Java Syntax


Below is some sample Java Source code that starts with identifying the fundamental set of rules required to code in Java.

public class FirstProgram{
	public static void main(String args[]){
		printHelloWorld();
	}
	
	public static void printHelloWorld(){
		System.out.println("Hello World!");
	}
}

Here are the main rules to consider when coding in Java:

  • Class name: The first letter of the class name should conventionally be upper case and every subsequent word’s first letter should also be uppercase. This convention is known as Pascal Case. The class acts as a template for an object. Don’t worry if you do not know what this means, it will be covered in the Advanced Java section.

  • Method name: A method acts as the behaviour of the class and is where all actions are executed along with any logical operations or data manipulation. The name of the method should conventionally start with a lower case letter and the subsequent word’s first letters should be upper case. This convention is known as Camel Case. An example of a method above would be printHelloWorld().

  • Curly braces: The beginning of a class and method should be started with the opening curly brace { and the end of the class or method should be terminated with a closing curly brace }. This helps the compiler identify what method or class each block of code belongs to.

  • Statement terminal: Each line that is not accompanied with a curly brace at the end must be terminated with a semi colon ; as part of Java syntax rules. Otherwise it will produce a compilation error which is when the code does not compile due to an error at compilation.

  • Identifiers: These are the names given to variables, classes and methods, such as the method printHelloWorld in the program above. They must begin with a letter, the $ character or the _ character. Also they cannot be a Java keyword.

  • Case Sensitivity: This means that printHelloWorld() and PrintHelloWorld() will result in different outputs in the program above. The latter will produce an error because there is no method called PrintHelloWorld(). This is because the compiler will look for the exact equivalent in terms of a variable or method name so it is important to remember to check this to avoid errors.



To demonstrate the importance of syntax, below are examples of compilation errors that occur when curly braces are missing or statement terminal semi-colons are missing:

	public static void main(String args[]){
